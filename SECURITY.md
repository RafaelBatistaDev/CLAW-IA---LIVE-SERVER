# Relatório de Segurança

## Divulgar Vulnerabilidades com Responsabilidade

Se você descobrir uma vulnerabilidade de segurança, **não abra uma issue pública**. 

Por favor, envie um email para: **rafaelbatistadev@outlook.com.br** com:
- Descrição da vulnerabilidade
- Passos para reproduzir
- Possível impacto
- Solução sugeria (se tiver)

## Políticas de Segurança

## Práticas de Segurança

Este projeto segue estas práticas:
- Validação rigorosa de entrada
- Sanitização de saída
- Princípio do menor privilégio
- Testes de segurança regulares
- Atualizações proativas de dependências
